# DEMON — Self-Evolving AI-Agent v1.0.0

---

## О проекте

**DEMON (DemonShell)** — это автономный многопоточный ИИ-агент для управления операционной системой Windows, функционирующий на базе локальной языковой модели **Ollama (Qwen2.5:7b)**. 

Проект оснащен графическим интерфейсом на PyQt6 с динамической RGB-подсветкой, работает в фоновом режиме, скрываясь в системном трее, управляется горячей клавишей `F4` и интегрируется в контекстное меню проводника Windows (ПКМ). Агент способен самостоятельно разрабатывать новые программные модули, изолировать их и динамически компилировать в оперативную память устройства.

---

## Ключевые возможности

*   **Многопоточность (Isolated Threads):** Пул из 4 независимых воркеров позволяет агенту параллельно выполнять ресурсоемкие задачи без блокировки графического интерфейса и смешивания контекста разных сессий.
*   **Самоэволюция (Plugin Architecture):** Архитектура поддерживает автоматическое написание новых скриптов на Python, их изоляцию в директории `Plugins` и динамический импорт в ОЗУ с автоматической генерацией JSON-схем для функционального вызова (Ollama Function Calling).
*   **Валидатор обработки исключений (Fallback-компилятор):** При попытке модели передать сырой Markdown-код вместо вызова инструмента, ядро перехватывает текстовые данные регулярными выражениями и принудительно отправляет SQL-запросы или скрипты на компиляцию.
*   **Фоновый режим:** Приложение функционирует без вызова консольных окон `cmd.exe` с использованием интерпретатора `pythonw.exe`. Вызов интерфейса осуществляется по триггеру горячей клавиши.
*   **IPC-сервер (Контекстное меню):** При запуске через контекстное меню проводника скрипт считывает выделенные данные через системный буфер обмена и передает текстовый поток по сокетам TCP (`127.0.0.1:9999`) в активный процесс ядра.

## Структура экосистемы

При первичном запуске приложение разворачивает изолированную инфраструктуру каталогов по пути `C:\Users\Имя_Пользователя\Desktop\DemonShell`:

```text
DemonShell/
├── Databases/          # Локальные базы данных SQLite3
├── Projects/           # Создаваемые проекты со своими изолированными средами .venv
├── Reports/
│   ├── Word/          # Сгенерированные документы .docx (Times New Roman, 14pt)
│   └── Excel/         # Сгенерированные отчеты .xlsx и таблицы данных
├── Plugins/            # Изолированный каталог для динамических плагинов
├── Downloads/          # Скачанные медиафайлы и веб-изображения
├── GeneratedMedia/     # Изображения, сгенерированные через нейросетевое API Pollinations
└── Notes/              # Текстовые заметки и логи
```

## Архитектура программных модулей

*   `main.py` — Отвечает за анимацию интерфейса, сокет-сервер (IPC), регистрацию глобальных горячих клавиш, работу системного трея (`pystray`) и таймер подсветки рамок.
*   `brain.py` — Выступает оркестратором логики взаимодействия со слоем ИИ, управляет сообщениями потоков, валидирует вызовы функций и обрабатывает сценарии Fallback.
*   `tools.py` — Набор низкоуровневых системных утилит Windows (управление медиаплеером, захват экрана через `ImageGrab`, чтение телеметрии через `psutil`, диспетчеризация планировщика задач `schtasks`, тихая установка пакетов через `pip` и менеджер плагинов).

## Инструкция по развертыванию

### Системные требования
1. Операционная система Windows (запуск строго с правами администратора для модификации реестра и планировщика задач).
2. Запущенный локальный сервис [Ollama](https://ollama.com) с загруженной моделью:
   ```bash
   ollama run qwen2.5:7b
   ```

### Установка и запуск
1. Клонируйте репозиторий проекта:
   ```bash
   git clone https://github.com
   cd DEMON
   ```
2. Установите необходимые зависимости:
   ```bash
   pip install PyQt6 ollama pystray keyboard requests pandas openpyxl python-docx psutil pillow duckduckgo_search
   ```
3. Выполните инициализацию (первый запуск зарегистрирует службу в автозагрузке и создаст ключи в реестре):
   ```bash
   python main.py
   ```
4. Используйте клавишу `F4` для открытия и скрытия консоли управления.
